{{ _("Délimiter les contraintes de construction") }}

{% trans trimmed %} Certaines cartes demandent une construction adaptée pour être plus efficaces. Il faut alors se demander si l'impact de ces cartes dans le plan de jeu est suffisant pour les jouer. {% endtrans %}

{% trans trimmed %} Par exemple, il est possible de jouer {{ blood_moon }} et {{ tainted_pact }} dans le même deck. Il faut cependant pouvoir construire en tenant compte des différentes contraintes. {% endtrans %}

{% trans trimmed %} Voici une liste de quelques carte demandant une adaptation du choix des cartes : {% endtrans %}

  • {% trans trimmed %} {{ blood_moon }} impose de jouer autant de sources non-rouges que possible sous forme de terrains basiques. Inversement, il faut un minimum de sources rouges sous forme de terrains basiques ; {% endtrans %}
  • {% trans trimmed %} {{ tainted_pact }} demande de ne jouer que des cartes différentes. Cela signifie qu'il faut jouer au plus un terrain basique de chacune des couleurs du deck et leur version neigeuse ; {% endtrans %}
  • {% trans trimmed %} La zone de commandement impose un nombre minimum de terrains pour jouer le ou les commandants. Par exemple, il faut 44 terrains pour jouer de manière régulière {{ archelos_lagoon_mystic }} sans pioche supplémentaire et sans accélération de mana. {% endtrans %}

{{ _("Les trois étape de la construction de deck") }}

{% trans trimmed %} Construire un deck optimal nécessite de passer par trois étapes d'affinage. {% endtrans %}

{{ _("Définir une stratégie") }}

{% trans trimmed %} Il convient à ce moment-là de compléter le deck avec le maximum de cartes synergiques avec le plan de jeu. Une fois le deck complété, une phase de test est nécessaire pour déterminer si le deck se joue comme cela a été anticipé. {% endtrans %}

{% trans trimmed %} Si ce n'est pas le cas, deux options sont disponibles : arrêter de construire le deck ou redéfinir le squelette et rebroder autour. Si cela fonctionne, il faut passer à l'étape suivante. {% endtrans %}

{{ _("Définir une tactique") }}

{% trans trimmed %} La première phase de test a permis de déterminer que le deck valait le coup d'être travaillé davantage et c'est cette phase qui correspond à l'étape la plus longue : quels sont les problèmes que rencontre le deck actuellement ? Quelles sont les cartes qui n'ont pas fonctionné du tout lorsqu'elles ont été jouées ? {% endtrans %}

{% trans trimmed %} L'identification des cartes dites « inutiles » permet de libérer de la place dans le deck pour y ajouter des solutions aux problèmes rencontrés lors de la première phase de tests. {% endtrans %}

{% trans trimmed %} Il faut ensuite faire des tests intensifs et extensifs comme détaillés dans la section suivante. {% endtrans %}

{{ _("Valider les tactiques par rapport à la stratégie") }}

{% trans trimmed %} Il faut garder en tête qu'une action ne doit pas forcément aller dans le sens de la stratégie mais ne doit jamais aller à l’opposé. Cette étape permet de confronter les cartes choisies dans l'étape précédente à leur utilité dans le deck. {% endtrans %}

{% trans trimmed %} C'est également dans cette étape que sont ajoutées des cartes hors plan de jeu mais qui permettent au deck de conserver une présence et un niveau de menace élevé contre les autres decks. Il s'agit de cartes plutôt défensives qui sont rentrées à ce stade de la construction. {% endtrans %}

{{ _("Tester une carte")}}

{% trans trimmed loi_hypergeometrique=_("la loi hypergéométrique (à venir)") %} Pour tester une carte, il ne faut pas attendre de la piocher et de la jouer. En effet, les probabilités de piocher cette carte dans un deck de 98 à 99 cartes est faible et suit {{ loi_hypergeometrique }}. {% endtrans %}

{% trans trimmed %} Il vaut mieux considérer la carte comme étant en zone de commandement et de se poser régulièrement les questions suivantes : {% endtrans %}

  • {% trans trimmed %} Si la carte est piochée ce tour-ci, sera-t-elle jouable ? résoudra-t-elle la situation ? {% endtrans %}
  • {% trans trimmed %} Si la carte piochée est un tuteur, est-ce que la carte testée serait le meilleur choix ? {% endtrans %}
  • {% trans trimmed %} Est-ce que la carte, si elle est piochée au prochain tour, apporte une solution à la situation présente ? {% endtrans %}

{{ _("Tester et analyser les parties") }}

{% trans trimmed %} Tester le deck revient à jouer des parties. Il faut donc analyser ces parties pour comprendre ce qui a fonctionné et ce qui n'a pas fonctionné. Cela se traduit en deux étapes : {% endtrans %}

  1. {{ _("Garder une trace écrite de chaque partie avec les informations suivantes :") }}
    • {{ _("Qui a commencé ?") }}
    • {{ _("Qui a mulligané et combien de fois ?") }}
    • {{ _("Qui a gagné ?") }}
    • {{ _("Quel est le taux de victoire du deck ?") }}
  2. {{ _("Etudier le rythme de la partie et les choix effectués :") }}
    • {{ _("Quels choix ont fait gagner un avantage dans la partie et pourquoi ?") }}
    • {{ _("Quelles cartes ont permis de gagner un avantage et pourquoi ?") }}
    • {{ _("Quelles cartes ont été décevantes dans la partie et pourquoi ?") }}
    • {{ _("Quelles cartes aurait-il fallu piocher et pourquoi ?") }}
    • {{ _("Quelles cartes n'aurait-il pas fallu piocher et pourquoi ?") }}

{{ _("Discuter de ses retours") }}

{% trans trimmed %} Armé de tous ces retours sur chaque partie, il faut comparer son analyse avec son adversaire dans un premier temps puis avec un groupe extérieur à la partie qui permettra de juger si les ressentis à chaud étaient corrects. {% endtrans %}

{% trans trimmed %} L'avantage est de consolider les retours faits sur les parties et aussi de confronter sa propre méthode d'analyse à celle d'autres personnes pour affûter son regard et supprimer au fur et à mesure les différents biais qui sont ainsi révélés. {% endtrans %}

{{ _("Définir des critères universels d'acceptation") }}

{% trans trimmed %} Lors de la construction du deck, il faut se fixer des limites qu'il ne faut pas dépasser en terme de coût de mana pour un effet donné. {% endtrans %}

{% trans trimmed spot_removal=link("/cartes-thematiques/les-spot-removals", "spot removal") %} Par exemple, il est admis qu'un {{ spot_removal }} coûte 2 manas. En cas de surcoût, les avantages fournis doivent être justifiés et conséquents, au risque d'avoir un deck peu efficace. {% endtrans %}

{% trans trimmed %} De même, dans des decks agressifs, il est admis qu'une créature doit pouvoir infliger autant de dégâts que son coût en mana, parfois plus mais rarement moins. {% endtrans %}
